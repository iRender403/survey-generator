## 面试题：实现一个可开关的调试日志系统
### 场景描述
你在开发一个低代码平台，需要在多个组件中打印调试信息。但发现：

1. console.log 散落各处，难以管理
2. 有时想关掉所有日志，有时只想看某个模块的日志
3. 生产环境不能有任何调试输出
### 需求
实现一个 trace 函数，支持以下功能：

基础功能

- 可以按模块分类输出（如 'material', 'redux', 'render'）
- 可以全局开启/关闭
- 可以只开启特定模块
进阶功能

- 支持不同级别（debug/info/warn/error），用不同颜色区分
- 显示时间戳
- 支持分组折叠，避免控制台刷屏
使用方式

```
// 开启所有模块
trace('material', '组件被点击', { id: 1 });
// 输出：[09:15:32][material] 组件被点击 { id: 
1 }

// 只开启 material 模块时，redux 的日志不输出
trace('redux', '状态更新', state); // 无输出
```
### 追问
1. 如何设计配置，让使用者可以方便地开启/关闭？（考虑环境变量、URL参数等）
2. 如果未来要把日志发送到服务器而不是控制台，你的设计需要怎么改？
3. 如何确保生产环境零开销？（即使代码里有 trace 调用，也不执行任何逻辑）
### 时间
30-40 分钟，可以只写核心逻辑，不用完整实现所有功能。
