# 项目 Bug 记录（给初学者看的）

记录开发过程中遇到的问题和解决方法，避免以后踩同样的坑。

## Bug 1：JSX 文件必须用 .tsx 后缀

### 报错现象
在 `src/routers/index.ts` 文件中，使用 `<App />` 这种 JSX 语法时，TypeScript 报错。

### 原因
TypeScript 规定：**只要文件里用了 JSX 语法（比如 `<div>`、`<Component />`），文件后缀必须是 `.tsx`，不能是 `.ts`**。

### 解决方法
把 `src/routers/index.ts` 重命名为 `src/routers/index.tsx`。

### 记忆口诀
> 有尖括号 `<>`，就用 `.tsx`

## Bug 2：组件函数返回的不是 JSX

### 报错现象
```
"EditorView"不能用作 JSX 组件。
其类型"() => typeof Editor"不是有效的 JSX 元素类型。
```

### 原因
组件代码写错了：
```tsx
// ❌ 错误的写法
export default function Editor():typeof Editor{
  return (
    Editor  // 返回的是函数本身，不是 JSX 元素！
  )
}
```

React 组件必须返回 JSX 元素（也就是类似 HTML 的代码），但上面的代码返回的是 `Editor` 这个函数本身。

### 解决方法
```tsx
// ✅ 正确的写法
export default function EditorView() {
  return (
    <div>Editor View</div>  // 返回 JSX 元素
  )
}
```

### 记忆口诀
> 组件函数要返回 HTML 样子的东西（JSX），别返回函数名

## Bug 3：类型导入要用 `import type`

### 报错现象
```
"RouteObject"是一种类型，必须在启用 "verbatimModuleSyntax" 时使用仅类型导入进行导入。
```

### 原因
项目的 `tsconfig.app.json` 配置了一个选项：
```json
"verbatimModuleSyntax": true
```

这个选项的意思是：**类型和值要分开导入**。

- `createBrowserRouter` 是**值**（运行时实际存在的函数）
- `RouteObject` 是**类型**（编译时检查用的，运行时不存在）

### 解决方法
```tsx
// ❌ 错误的写法
import { createBrowserRouter, RouteObject } from 'react-router-dom';

// ✅ 正确的写法
import { createBrowserRouter, type RouteObject } from 'react-router-dom';
```

在类型前面加 `type` 关键字，告诉 TypeScript："这是一个类型，编译完就删掉"。

### 记忆口诀
> 从库导入东西时，如果是类型（比如 `RouteObject`、`Props`），前面加 `type`

### 快速判断方法
不确定是不是类型？看它在代码中怎么用：
```tsx
// 用在 : 后面的是类型
const routes: RouteObject[] = []  // RouteObject 是类型

// 直接调用的是值
createBrowserRouter(routes)  // createBrowserRouter 是值
```

## 总结：三个避坑要点

| 要点 | 说明 |
|------|------|
| 1. 文件后缀 | 用 JSX 就用 `.tsx` |
| 2. 组件返回 | 必须返回 JSX，别返回函数名 |
| 3. 类型导入 | 类型前面加 `type` |
